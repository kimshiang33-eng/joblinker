# JobLinker interface direction

- Job-content-first utility layout; avoid a marketing-style hero on the public board.
- English is primary, with a visible BM switch that changes all interface and job copy.
- WhatsApp is the primary job action and must remain visible without opening a detail page.
- Use solid blue accents, white and light grey surfaces, thin borders, 5–6px radii, and almost no shadow.
- Keep mobile information dense but readable: title, company, location, salary, verification, then WhatsApp.
- Prefer plain controls and real interface labels over decorative cards, gradients, oversized headings, or promotional copy.
- Employer authentication uses the same flat borders and restrained blue accents; the form stays primary and the supporting value proposition remains concise.
- Employer dashboard surfaces operational metrics and job actions without decorative charts; desktop uses a compact sidebar while mobile keeps a single-column task flow.
- Job posting is a three-step guided flow with a live listing preview, local draft persistence, and direct WhatsApp contact validation.
- Admin operations prioritize the pending-review queue, clear listing states, reversible Featured controls, and simple views-to-WhatsApp performance metrics.
- Public job detail pages keep salary, location and WhatsApp above the fold, use a fixed mobile application bar, and place complete responsibilities, requirements and benefits below the primary action.
